Contribute

Anyone can contribute to Gecode/R by for instance pointing out flaws in the documentation, requesting features, giving feedback, sending patches, reporting bugs and being active on the mailing lists.

Interact

Tell us what you think of Gecode/R. Tell us what could be better, what should be changed, what is hard to understand. We want to know so that we can fix it.

Use gecoder-users for such feedback (just send a mail there if you want to, there’s no need to sign up). Subscribe to that mailing list and be helpful to contribute even more.

Report Bugs

Report bugs to the bug tracker. Make sure that you include enough information for others to be able to reproduce the bug.

Submit Patches

Send your patches to the patch tracker, preferably as diffs against the current trunk. If you make a habit of sending good patches then we will just save us the hassle and give you direct commit access.

Add Feature Requests

If you have an idea for a feature, but can’t implement it yourself, then send it to the feature request tracker to let someone else have a go at it.
